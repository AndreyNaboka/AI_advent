# Goal-driven файловый ассистент

## Что реализовано

`file_assistant.py` принимает цель на естественном языке и сам выбирает
необходимые операции с файлами. Прямые пути исходников пользователю указывать не
нужно. Ассистент вызывает `mcp_filesystem_server.py` и использует четыре MCP
инструмента:

- `list_project_files` — найти подходящие файлы;
- `search_text` — выполнить поиск сразу по проекту;
- `read_file` — прочитать и проанализировать найденные файлы;
- `write_file` — создать или обновить артефакт и вернуть unified diff.

Запись ограничена каталогами `docs/`, `assistant_outputs/`, `adrs/` и корневыми
Markdown-файлами. Абсолютные пути и `..` запрещены. Каталоги `.git`, виртуальные
окружения, сборочные артефакты и предыдущие отчёты не участвуют в поиске.

## Сценарий 1: найти использования компонента

Пользователь задаёт цель, а не список файлов:

```bash
venv312/bin/python file_assistant.py \
  --goal 'Найди все использования компонента "MCPCRMClient" и подготовь отчёт'
```

Ассистент самостоятельно:

1. ищет `MCPCRMClient` в Python, JavaScript, TypeScript и Markdown;
2. читает каждый найденный файл;
3. классифицирует совпадения как определение, импорт, создание или ссылку;
4. сохраняет `assistant_outputs/usage_mcpcrmclient.md`;
5. выводит unified diff созданного или изменённого отчёта.

## Сценарий 2: обновить документацию по коду

```bash
venv312/bin/python file_assistant.py \
  --goal 'Обнови документацию по структуре Python-модулей проекта'
```

Ассистент находит production Python-файлы, читает их, анализирует AST и создаёт
`docs/generated_code_inventory.md` со списком module docstrings, классов,
функций и импортов.

## Dry-run и повторяемость

Проверка без записи:

```bash
venv312/bin/python file_assistant.py \
  --goal 'Обнови документацию по структуре Python-модулей проекта' \
  --dry-run
```

При первом запуске выводится diff. Повторный запуск на неизменном проекте должен
сообщить `Изменения: нет` и `Diff пуст: результат уже актуален`.

Для изолированной демонстрации можно передать корень fixture-проекта:

```bash
venv312/bin/python file_assistant.py \
  --root tests/fixtures/file_assistant/project \
  --goal 'Найди все использования компонента "PaymentGateway" и подготовь отчёт'
```

## Автоматические тесты

```bash
venv312/bin/python -m unittest tests.test_file_assistant -v
```

Тесты проверяют работу минимум с тремя файлами, MCP handshake, поиск, чтение,
запись, diff, идемпотентность, dry-run и блокировку path traversal.
