# Помощник по документации проекта

## Назначение

Project Help Assistant отвечает на вопросы о структуре AI Advent на основании
локальной документации. Источниками RAG служат корневой `README.md` и файлы из
папки `docs/`. Если релевантный фрагмент не найден, помощник сообщает об этом и
не придумывает сведения о проекте.

## Поток данных

Команда `/help <вопрос>` передаёт вопрос в `ProjectHelpAssistant`. Компонент
`ProjectRagIndex` читает документацию, делит Markdown по заголовкам и ранжирует
чанки BM25-подобным лексическим поиском. Из лучших фрагментов строится короткий
ответ со списком путей и разделов-источников.

Параллельно `MCPProjectClient` запускает `mcp_project_server.py` по stdio,
выполняет MCP handshake и вызывает `git_branch`. Полученная текущая ветка
добавляется в ответ как живой контекст проекта.

## MCP API

MCP-сервер предоставляет только read-only инструменты:

- `git_branch` без аргументов возвращает поле `branch` с текущей Git-веткой;
- `list_files` принимает необязательный `limit` и возвращает отслеживаемые файлы;
- `git_diff` принимает необязательный `max_chars` и возвращает незакоммиченный diff.

Сервер использует JSON-RPC 2.0 и MCP protocol version `2024-11-05`. Git-команды
запускаются без shell и всегда ограничены корнем текущего проекта.

## Ограничения минимальной версии

Встроенный индекс полностью локальный и не требует сервисов, но использует
лексический поиск. Для семантического поиска основной проект также содержит
индексатор Ollama + Qdrant в папке `rag_indexer_tool/`.
